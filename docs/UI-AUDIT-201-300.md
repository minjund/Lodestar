# UI 버튼 전수 점검 — 신규 버그 수정·기능 개선 201–300

기존 [`UI-AUDIT-100.md`](./UI-AUDIT-100.md)의 1–100과 [`UI-AUDIT-101-200.md`](./UI-AUDIT-101-200.md)의 101–200은 제외 기준으로 고정한다. 아래 항목은 전역 버튼·입력 제어의 상태 복구, 포커스, 작은 화면 조작, 저장소 안전성을 새로 분리한 항목이며, 코드와 검증에 반영된 뒤에만 `[x]`로 바꾼다.

모든 항목은 현재 파일과 기대 동작이 명확해 `auto-fixable`로 분류했다. 심각도는 입력·포커스 손실과 저장소 복구를 `high`, 제어 상태 전달과 모바일 조작을 `medium`, 마찰 감소를 `low`로 두었다.

## 전역 제어 상태와 입력 방식 (201–220)

201. [x] (`medium`) 전역 품질 설정을 버전이 있는 로컬 저장 형식으로 분리한다.
202. [x] (`high`) 손상된 품질 설정 저장값은 기본값으로 복구한다.
203. [x] (`medium`) 마지막 입력 방식이 키보드인지 포인터인지 상태에 기록된다.
204. [x] (`medium`) 키보드 조작으로 전환되면 본문에 키보드 모드 클래스를 적용한다.
205. [x] (`medium`) 포인터 조작으로 전환되면 키보드 모드 클래스를 해제한다.
206. [x] (`low`) 입력 방식 상태는 다음 실행에도 보존된다.
207. [x] (`medium`) 품질 설정의 모션 선호값을 본문 데이터 속성으로 반영한다.
208. [x] (`medium`) 품질 설정의 밀도 선호값을 본문 데이터 속성으로 반영한다.
209. [x] (`low`) 품질 설정 기본값을 한 함수에서 재사용한다.
210. [x] (`medium`) 품질 설정 저장 실패를 복구 가능한 오류로 보고한다.
211. [x] (`medium`) 전역 제어 보강은 중복 설치되지 않는다.
212. [x] (`medium`) 전역 제어 보강 설치 후 기존 제어 요소를 즉시 스캔한다.
213. [x] (`medium`) 새로 렌더링되는 제어 요소도 MutationObserver로 보강한다.
214. [x] (`low`) 제어 보강 재스캔은 animation frame으로 묶어 과도한 반복을 줄인다.
215. [x] (`medium`) disabled 속성이 바뀌면 제어 상태를 다시 계산한다.
216. [x] (`medium`) aria-disabled 속성이 바뀌면 제어 상태를 다시 계산한다.
217. [x] (`low`) 클래스 변경으로 크기나 라벨이 바뀌어도 제어 상태를 다시 계산한다.
218. [x] (`low`) 제목과 접근 가능한 이름 변경도 보강 대상으로 잡는다.
219. [x] (`medium`) 뷰포트 폭을 mobile/tablet/desktop 상태로 분류한다.
220. [x] (`medium`) 창 크기 변경 후 뷰포트 분류를 갱신한다.

## 버튼 라벨·터치 타깃·눌림 상태 (221–240)

221. [x] (`high`) 아이콘 버튼에 접근 가능한 이름이 없으면 기존 텍스트와 title에서 보완한다.
222. [x] (`high`) 상단 아이콘 액션에 접근 가능한 이름이 없으면 보완한다.
223. [x] (`high`) 닫기 버튼에 접근 가능한 이름이 없으면 보완한다.
224. [x] (`medium`) 모든 버튼에 품질 제어 데이터 표식을 추가한다.
225. [x] (`medium`) role=button 요소에도 품질 제어 데이터 표식을 추가한다.
226. [x] (`low`) input 요소에도 품질 제어 데이터 표식을 추가한다.
227. [x] (`low`) select 요소에도 품질 제어 데이터 표식을 추가한다.
228. [x] (`low`) textarea 요소에도 품질 제어 데이터 표식을 추가한다.
229. [x] (`medium`) type이 없는 button은 기본 submit으로 새지 않도록 button 타입을 지정한다.
230. [x] (`medium`) 필수 input은 aria-required를 함께 노출한다.
231. [x] (`medium`) 필수 textarea는 aria-required를 함께 노출한다.
232. [x] (`medium`) 필수 select는 aria-required를 함께 노출한다.
233. [x] (`medium`) 40px보다 작은 버튼은 터치 타깃 보강 상태로 표시한다.
234. [x] (`low`) 작은 터치 타깃은 가상 hit area를 제공한다.
235. [x] (`medium`) pointerdown 시 버튼 눌림 상태를 데이터 속성으로 표시한다.
236. [x] (`medium`) pointerup 시 버튼 눌림 상태를 해제한다.
237. [x] (`medium`) pointercancel 시 버튼 눌림 상태를 해제한다.
238. [x] (`medium`) pointerleave 시 버튼 눌림 상태를 해제한다.
239. [x] (`medium`) Space 키로 버튼을 누를 때도 눌림 상태를 표시한다.
240. [x] (`medium`) Enter 키로 버튼을 누를 때도 눌림 상태를 표시한다.

## 비활성·오버플로·폼 복구 (241–260)

241. [x] (`medium`) keyup 후 키보드 눌림 상태를 해제한다.
242. [x] (`medium`) disabled 버튼은 품질 비활성 상태로 표시한다.
243. [x] (`medium`) aria-disabled 버튼은 품질 비활성 상태로 표시한다.
244. [x] (`low`) 다시 활성화된 버튼은 품질 비활성 상태를 제거한다.
245. [x] (`medium`) 비활성 버튼에 설명이 없으면 기본 비활성 사유를 제공한다.
246. [x] (`medium`) 비활성 사유 문구를 한국어로 제공한다.
247. [x] (`medium`) 비활성 사유 문구를 영어로 제공한다.
248. [x] (`medium`) 비활성 사유 문구를 중국어로 제공한다.
249. [x] (`low`) 제어 라벨 후보는 aria-label을 우선한다.
250. [x] (`low`) 제어 라벨 후보는 title을 두 번째로 사용한다.
251. [x] (`low`) 제어 라벨 후보는 보이는 텍스트를 세 번째로 사용한다.
252. [x] (`low`) 제어 라벨 후보는 i18n 키와 id까지 안전하게 fallback한다.
253. [x] (`medium`) 라벨 후보의 연속 공백을 하나로 정규화한다.
254. [x] (`medium`) 라벨 후보를 180자로 제한해 과도한 title을 막는다.
255. [x] (`medium`) 넘치는 버튼 라벨에는 title을 자동 보완한다.
256. [x] (`medium`) 넘치는 내비게이션 항목에는 title을 자동 보완한다.
257. [x] (`medium`) 넘치는 메타 칩에는 title을 자동 보완한다.
258. [x] (`low`) 넘치는 세션 카드 제목에는 title을 자동 보완한다.
259. [x] (`low`) 넘치는 터미널 세션 카드에는 title을 자동 보완한다.
260. [x] (`medium`) 창 크기 변경 후 오버플로 title을 다시 계산한다.

## 입력 오류 회복과 고급 설정 보존 (261–280)

261. [x] (`high`) 사용자가 잘못된 입력을 고치면 aria-invalid를 자동 해제한다.
262. [x] (`high`) 사용자가 폼 입력을 고치면 보이는 폼 오류를 자동 숨긴다.
263. [x] (`medium`) 빈 값 입력은 오류 해제를 성급하게 하지 않는다.
264. [x] (`medium`) 모델 입력은 blur 시 앞뒤 공백을 제거한다.
265. [x] (`medium`) tmux 작업 묶음 이름은 blur 시 앞뒤 공백을 제거한다.
266. [x] (`medium`) tmux 시작 폴더는 blur 시 앞뒤 공백을 제거한다.
267. [x] (`medium`) 새 작업 고급 설정 펼침 상태를 품질 설정에 보존한다.
268. [x] (`medium`) 새 작업 고급 설정은 다시 열 때 이전 펼침 상태를 복원한다.
269. [x] (`medium`) 고급 설정 toggle 후 저장 실패를 복구 가능한 오류로 보고한다.
270. [x] (`low`) 줄어든 모션 설정은 전체 전환 시간을 최소화한다.
271. [x] (`low`) 줄어든 모션 설정은 전체 애니메이션 시간을 최소화한다.
272. [x] (`low`) 줄어든 모션 설정은 반복 애니메이션을 1회로 제한한다.
273. [x] (`low`) 줄어든 모션 설정은 부드러운 스크롤을 끈다.
274. [x] (`medium`) 키보드 모드에서는 focus-visible 윤곽선을 더 강하게 표시한다.
275. [x] (`medium`) 모든 품질 제어는 터치 조작을 manipulation으로 제한한다.
276. [x] (`low`) 눌림 상태는 밝기와 위치 변화로 즉시 보인다.
277. [x] (`low`) 비활성 상태는 not-allowed 커서로 드러난다.
278. [x] (`low`) compact 밀도 상태에서는 세션 격자 간격을 줄인다.
279. [x] (`low`) compact 밀도 상태에서는 터미널 레이아웃 간격을 줄인다.
280. [x] (`medium`) 모바일 뷰포트에서는 품질 모달 모서리 처리를 화면 하단에 맞춘다.
