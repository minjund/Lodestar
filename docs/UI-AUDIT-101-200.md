# UI 버튼 전수 점검 — 신규 버그 수정·기능 개선 101–200

기존 [`UI-AUDIT-100.md`](./UI-AUDIT-100.md)의 1–100은 제외 기준으로 고정한다. 아래 항목은 실제 Electron UI, 접근성 트리, 112개 조작 선택자의 3회 전수 실행에서 새로 분리한 항목이며, 코드와 검증에 반영된 뒤에만 `[x]`로 바꾼다.

모든 항목은 현재 파일과 기대 동작이 명확해 `auto-fixable`로 분류했다. 심각도는 키보드·입력 손실·오류 복구를 `high`, 상태 전달·탐색을 `medium`, 편의 기능을 `low`로 두었다.

## 빠른 이동·전역 버튼 계약 (101–120)

101. [x] (`medium`) `Cmd/Ctrl+K`로 빠른 이동 창을 연다.
102. [x] (`low`) 빠른 이동 창에서 주요 화면과 자주 쓰는 작업을 한 번에 찾는다.
103. [x] (`medium`) 빠른 이동 검색어에 맞춰 명령 목록을 즉시 거른다.
104. [x] (`medium`) 빠른 이동 목록을 위·아래 방향키로 순환한다.
105. [x] (`medium`) 빠른 이동 목록에서 Home·End로 처음과 끝에 이동한다.
106. [x] (`medium`) 빠른 이동 창에서 Enter로 선택 명령을 실행한다.
107. [x] (`medium`) 빠른 이동 창에서 Escape로 닫고 원래 초점으로 돌아간다.
108. [x] (`medium`) 빠른 이동 배경을 눌러 닫아도 원래 초점으로 돌아간다.
109. [x] (`high`) 빠른 이동 입력이 현재 항목을 `aria-activedescendant`로 알린다.
110. [x] (`medium`) 빠른 이동 결과가 없을 때 상태 메시지를 알린다.
111. [x] (`low`) `?` 키로 키보드 단축키 도움말을 연다.
112. [x] (`low`) 상단에 단축키 도움말을 여는 전용 버튼을 추가한다.
113. [x] (`medium`) 단축키 도움말을 Escape로 닫고 초점을 복원한다.
114. [x] (`medium`) 단축키 도움말 배경을 눌러 닫고 초점을 복원한다.
115. [x] (`high`) 새 전역 대화상자도 본문을 `inert`로 격리하고 초점을 가둔다.
116. [x] (`medium`) 일곱 화면 이동 버튼을 실제 제어 영역과 `aria-controls`로 연결한다.
117. [x] (`medium`) 화면 이동 버튼의 접근 가능한 이름에 최신 개수를 동기화한다.
118. [x] (`low`) AI 재확인 버튼을 제공사 상태 영역과 연결한다.
119. [x] (`low`) 작업 폴더 추가 버튼을 작업 폴더 목록과 연결한다.
120. [x] (`low`) 업데이트 안내 버튼을 설정의 업데이트 패널과 연결한다.

## 검색·필터·작업 폴더 상태 보존 (121–140)

121. [x] (`medium`) 대시보드 조건을 버전이 있는 로컬 저장 형식으로 보존한다.
122. [x] (`medium`) 다시 열었을 때 검색어를 복원한다.
123. [x] (`medium`) 다시 열었을 때 복수 AI 필터를 복원한다.
124. [x] (`medium`) 다시 열었을 때 작업 폴더 필터를 복원한다.
125. [x] (`medium`) 다시 열었을 때 정렬 기준을 복원한다.
126. [x] (`high`) 손상되거나 오래된 저장값은 안전한 기본값으로 복구한다.
127. [x] (`medium`) 검색어를 240자로 제한해 과도한 렌더링을 막는다.
128. [x] (`low`) 검색 비교에서 연속 공백을 하나로 정규화한다.
129. [x] (`low`) 현재 결과 개수를 화면에 보이는 요약으로 표시한다.
130. [x] (`medium`) 검색·필터·정렬·더 보기 후 결과 요약을 즉시 갱신한다.
131. [x] (`medium`) 검색 입력에서 아래 방향키로 첫 결과 카드에 이동한다.
132. [x] (`medium`) 검색 결과가 하나면 Enter로 바로 상세를 연다.
133. [x] (`low`) 필터 때문에 빈 화면이 되면 ‘조건 지우기’ 버튼을 제공한다.
134. [x] (`low`) ‘조건 지우기’ 버튼은 실제 조건이 있을 때만 표시한다.
135. [x] (`medium`) 빈 화면의 ‘조건 지우기’가 모든 조건을 초기화하고 검색에 초점을 둔다.
136. [x] (`medium`) 작업 폴더 선택 목록에 명시적인 그룹 이름을 제공한다.
137. [x] (`medium`) AI 필터를 키보드 도구 모음으로 표현한다.
138. [x] (`medium`) AI 필터에는 한 개의 순회 Tab 정지점만 둔다.
139. [x] (`medium`) AI 현황 카드에도 한 개의 순회 Tab 정지점만 둔다.
140. [x] (`medium`) 조건 변경·초기화 직후 저장 상태를 실제 UI와 동기화한다.

## 새 AI 작업 작성·검증 (141–160)

141. [x] (`medium`) 새 작업 작성 상태를 버전이 있는 세션 초안으로 보존한다.
142. [x] (`high`) 작성 중인 요청 내용을 닫았다 다시 열어도 복원한다.
143. [x] (`high`) 작성 중인 작업 폴더를 닫았다 다시 열어도 복원한다.
144. [x] (`medium`) 직접 지정한 모델을 닫았다 다시 열어도 복원한다.
145. [x] (`high`) 파일 수정 허용 선택을 닫았다 다시 열어도 복원한다.
146. [x] (`medium`) 선택한 AI를 닫았다 다시 열어도 복원한다.
147. [x] (`high`) 손상된 초안 저장값은 입력을 막지 않고 기본값으로 복구한다.
148. [x] (`low`) 작성 중인 값을 한 번에 비우는 ‘초안 지우기’ 버튼을 추가한다.
149. [x] (`medium`) 초안 지우기 후 모든 작성 상태를 초기화하고 요청 입력에 초점을 둔다.
150. [x] (`medium`) 요청 입력을 실시간 글자 수와 `aria-describedby`로 연결한다.
151. [x] (`high`) 공백만 있는 요청을 유효한 필수 입력으로 통과시키지 않는다.
152. [x] (`high`) 공백만 있는 작업 폴더를 유효한 필수 입력으로 통과시키지 않는다.
153. [x] (`high`) 제출 검증 실패 시 첫 번째 잘못된 입력으로 초점을 옮긴다.
154. [x] (`medium`) 요청 공백 오류를 구체적인 메시지로 표시한다.
155. [x] (`medium`) 작업 폴더 공백 오류를 구체적인 메시지로 표시한다.
156. [x] (`medium`) 사용자가 입력을 고치기 시작하면 이전 실행 오류를 지운다.
157. [x] (`low`) 제출 버튼에 `Cmd/Ctrl+Enter` 단축키 정보를 노출한다.
158. [x] (`low`) 모델 입력의 브라우저 자동완성을 끈다.
159. [x] (`low`) 모델 입력의 맞춤법 교정을 끈다.
160. [x] (`high`) 모달 안에서 누르기 시작해 바깥에서 놓아도 실수로 닫히지 않게 한다.

178. [x] (`low`) 잘린 상세 제목의 전체 텍스트를 툴팁으로 제공한다.

180. [x] (`medium`) 상세 재개 버튼의 중복 클릭을 막고 진행 상태를 표시한다.
